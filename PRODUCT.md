# Product

## Register

product

## Users
PHP developers building or maintaining websites who need content discovered and accurately cited by AI-powered search engines (ChatGPT, Perplexity, Claude, Google AI Overviews). Typically mid-to-senior developers or agencies managing client sites. They know PHP, understand Composer, and want a library that works without requiring deep AI knowledge.

## Product Purpose
GeoOptimizer is a PHP library for Generative Engine Optimization (GEO) — structuring and presenting web content so AI search engines cite it accurately and rank it prominently. It generates llms.txt files, Schema.org structured data, content readiness scores, and citation tracking across platforms. Success means a developer can install it, run it against their site, and improve their AI search presence in under 30 minutes.

## Brand Personality
Precise, helpful, unpretentious. A tool that knows what it does and says so clearly. Tone is direct and technical but not terse — the kind of README you actually read to the end.

## Anti-references
- GitHub's design system (not the #0d1117 + #238636 palette)
- Generic SaaS landing pages (no hero metrics, gradient text, glass cards, startup template energy)
- AI-generated scaffold pages (no eyebrows on every section, no 01/02/03 section numbering, no cream/sand warm-neutral backgrounds)
- Heavy documentation portals (no sidebar-dominant layout, no nav-heavy structure for what is a single landing page)

## Design Principles
1. Show, don't tell. Features are demonstrated with real code, not described in marketing adjectives.
2. Trust developers. Assume the audience can read code. No hand-holding theater.
3. Minimal surface, maximum signal. The UI steps aside. Content and functionality are the product.
4. One identity, not a clone. Visual choices traceable to GeoOptimizer's own brand, not borrowed from a more famous tool.
5. Content earns its place. Nothing on the page that doesn't help a developer decide to use the library or understand how.

## Accessibility & Inclusion
WCAG 2.1 AA minimum. prefers-reduced-motion support. Body text ≥7:1 contrast on dark background. Keyboard-navigable with visible focus rings.
